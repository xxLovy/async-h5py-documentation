                              Why async h5py

                              Asynchronous I/O is becoming increasingly popular with the large amount of data access required by scientific applications. They can take advantage of an asynchronous interface by scheduling I/O as early as possible and overlap computation or communication with I/O operations, which hides the cost associated with I/O and improves the overall performance.

                              Here is the test of data write operation. When the order of magnitude reaches 108 We can see that the asynchronous function is over 4000 times faster than the synchronous function! Imagine how much performance can be improved for larger data!

                              Installing async h5py

                              1. Install and configure parallel HDF5 and vol-async connector(vol-async installation)
                              2. Install and configure async h5py(parallel h5py installation)

                              How to use async h5py

                              Set Environmental Variables

                              We should set environmental variables before we use async h5py code.

                              Linux:

                              xxxxxxxxxx
                              export LD_LIBRARY_PATH=$VOL_DIR/src:$HDF5_DIR/install/lib:$ABT_DIR/install/lib:$LD_LIBRARY_PATH
                              export HDF5_PLUGIN_PATH="$VOL_DIR/src"
                              export HDF5_VOL_CONNECTOR="async under_vol=0;under_info={}"
                              (optional)export LD_PRELOAD=$VOL_DIR/src/libh5async.so:$HDF5_DIR/install/lib/libhdf5.so:$ABT_DIR/install/lib/libabt.so

                              Code examples of async h5py

                              Eventset

                              An event set (ID) is an in-memory object that is created by the application and functions similar to a “bag” holding request tokens from one or more asynchronous I/O operations. Every asynchronous I/O operation must be associated with an event set

                              so first we should create an eventset for tracking async operations

                              xxxxxxxxxx
                              import h5py
                              es=h5py.Eventset()

                              The application can then use this event set ID (es_id) for all subsequent h5py I/O operations. One can also create multiple event set IDs (es_id) for different operations. Because the h5py functions are non-blocking when async VOL is enabled, the application should also wait for all asynchronous operations to finish before exiting, with:

                              xxxxxxxxxx
                              es.wait(timeout) #timeout can be any uint_64 number but should be large enough

                              after the wait operation we can check the the number of operations still in progress and flag if an operation in the event set failed with:

                              xxxxxxxxxx
                              es.num_in_progress #should be 0, if not some error occured
                              es.op_failed       #should be False, if not some error occured

                              The wait operation is also required before closing an event set, with:

                              xxxxxxxxxx
                              es.close()

                              File

                              After we create an eventset, we can use that to do something with file first.

                              first we should create a file

                              xxxxxxxxxx
                              #use async h5py to create a file
                              f = h5py.File("name.hdf5", "w", es=es)

                              and also if we want to use a HDF5 file we can open a file as well

                              xxxxxxxxxx
                              #use async h5py to open a file
                              f0 = h5py.File("name.hdf5", "r", es=es)

                              if we need to flush a file asynchronously we can use this

                              xxxxxxxxxx
                              #use async h5py to flush a file
                              f.flush()#or f.flush(es=es)

                              when we finish all the operations we can finally use f.close() to close a file asynchronously

                              xxxxxxxxxx
                              #use async h5py to close a file
                              f.close()

                              don't forget call es.wait(timeout) after we close a file asynchronously and before we close es

                              xxxxxxxxxx
                              es.wait(timeout) #timeout can be any uint_64 number but should be large enough
                              es.close()

                              Group

                              after we have a file, we can create a group in that file

                              xxxxxxxxxx
                              #use async h5py to create a group
                              grp = f.create_group("grp", es=es)

                              Dataset

                              dataset is an important part in h5py, it will carry the data of HDF5 file

                              first we should create a dataset

                              xxxxxxxxxx
                              #use async h5py to create a dataset
                              dset = f.create_dataset("dset", (10, 20), es=es)

                              and also if we want to do something with a dataset we can open a dataset as well

                              xxxxxxxxxx
                              #use async h5py to open a dataset
                              dset0 = f["dset"]

                              now we can write something in a dataset.

                              first we use numpy to create a numpy array and we can write that to the dataset we have.

                              xxxxxxxxxx
                              import numpy as np
                              data_write = np.arange(200, dtype=int)

                              and we can write that numpy array to the dataset we have.

                              xxxxxxxxxx
                              #use async h5py to write data
                              dset.write_direct(data_write.reshape(10, 20), es=es)

                              and if we want to read the data in dataset, we can use this to achieve that

                              xxxxxxxxxx
                              #use async h5py to read data
                              data_read = np.empty((10, 20), dtype=int)
                              dset.read_direct(data_read.reshape(10, 20), es=es)

                              h5py has provided a method "resize", and we can use that method to change the shape of a dataset, now we can do that asynchronously

                              xxxxxxxxxx
                              #use async h5py to resize dataset
                              dset = f.create_dataset('foo', (20, 30), maxshape=(20, 60), es=es)
                              dset.resize((20, 50), es=es)

                              Attribute

                              attribute is the official way to store metadata in HDF5 and we can create an attribute with:

                              xxxxxxxxxx
                              #attribute create
                              f.attrs["attr_file"] = 1
                              grp.attrs["attr_grp"] = 2
                              dset.attrs["attr_dset"] = 3

                              we should notice that the es_id we pass to the attribute operation is from the object we use(for example grp.attrs["attr_grp"] = 2 will use es_id from grp as the es_id we pass to the attribute operation) and we can modify that by changing the value of the object's es_id(for example we can use grp.es_id=es2 to change the es_id we pass to the attribute operation)

                              and if we want to change its value asynchronously, we can use modify to achieve that with:

                              xxxxxxxxxx
                              #attribute modify
                              f.attrs.modify("attr_file", 1)

                              Something to notice

                              When using async function to create a file, group or dataset, some of their function will be called asynchronously, if we don't want to use aysnc mode, we should set its es_id property to None. E.g:

                              xxxxxxxxxx
                              f = h5py.File("name", "w", es=es)
                              dset = f.create_dataset("dset", dtype=int, es=es)#dataset will be created asynchronously
                              dset.attr["dset"] = 2#the asynchronous function will be called and the es value will be the one when we create the dset. Which is dset.es_id

                              If we want this step to be synchronous, we need to use the following steps:

                              xxxxxxxxxx
                              dset.es_id=None
                              dset.attr["dset"] = 3#The async function will not be called at this point

                              This method allows us to automatically call its asynchronous function without passing in the parameter es when performing some operations. For example, after we create a file asynchronously, when the file is closed, we can use file.close() to call the asynchronous function. Note that this feature applies to all attribute operations which have an asynchronous version. Because we use attribute operations most of the time we will use such as dset.attrs[] or dset.attrs.modify(). Also some other functions that uses this methon are listed here:

                              • File: f.close(), f.flush(), f[]
                              • Group: grp[]
                              • Dataset: None
                              • Attribute: .attrs[], .attrs.modify()

                              An example

                              ​x
                              import h5py
                              #create an Eventset and a file
                              es_id=h5py.Eventset()
                              f = h5py.File("file.hdf5", "w", es=es_id)
                              #create a group under the file we create and then create a dataset under that
                              grp = f.create_group("grp", es=es_id)
                              dset = grp.create_dataset("dset", (10, 20), maxshape=(20, 60), es=es_id)
                              

                              import numpy as np
                              data_read = np.empty((10, 20), dtype=int)
                              data_write = np.arange(200, dtype=int)
                              

                              #now we can use write_direct
                              dset.write_direct(data_write.reshape(10, 20), es=es_id)
                              dset.read_direct(data_read.reshape(10, 20), es=es_id)
                              

                              dset.resize((20, 50), es=es_id)
                              f.attrs["file"] = 1
                              grp.attrs["grp"] = 2
                              dset.attrs["dset"] = 3
                              

                              # now we can verify the data we have
                              es_id.wait(1000000)
                              print("the attribute value of file: " + str(f.attrs["file"]))
                              print("the attribute value of group: " + str(grp.attrs["grp"]))
                              print("the attribute value of dataset: " + str(dset.attrs["dset"]))
                              f.attrs.modify("file", 4)
                              es_id.wait(1000000)
                              print("the attribute value of file(modified): " + str(f.attrs["file"]))
                              print("dset.shape(resized) = " + str(dset.shape))
                              

                              f.es_id=None
                              f.close()
                              es_id.wait(1000000)
                              es_id.close()

                              Limitation

                              • Now we can't read (H5Aread_async) or open (H5Aopen_by_name_async, H5Aopen_by_idx_async) an attribute asynchronous. Which means we can't use something like f.attrs["name"] to open an attribute or read its value.
                              • Now there is some problem with async file close function(H5Fclose_async) that we haven't solved yet. Under some circumstances some error will occur. For example when we create a group and the asynchronous close function will not work.
                              • Currently the async function only works with the first read/write operation. From the second read/write operation due to the H5Dget() called by h5py they are effectively synchronous.
